Warden — AI Treasury Agent

Autonomous treasury management powered by Tether WDK.
Enforces spending policies, auto-approves routine transfers,
escalates the ones that need human eyes, and logs every decision.

What Warden Does
🛡️

Policy Engine

Five rule types — blacklist, per-tx limit, whitelist multiplier, daily cap, hourly rate limit. Stateless, testable, deterministic.

🔑

WDK Integration

Uses WalletManagerEvm + account.transfer() from Tether WDK for self-custodial BIP-44 key derivation and EVM signing.

📋

Immutable Audit Log

Every APPROVE / REJECT / ESCALATE decision is persisted to SQLite with full context — recipient, amount, reason, timestamp.

🌐

REST API

Express server exposes /v1/transfer, /v1/balance, /v1/stats, and /v1/audit endpoints.

✅

45 Tests, 0 Failures

PolicyEngine (28 tests) and AuditLogger (17 tests) — using Node.js built-in node:test. No framework deps.

⚡

Zero Custodial Tradeoffs

Warden sits on top of WDK — not instead of it. WDK's self-custody guarantees remain intact; Warden only gates the signing layer.

Policy Rules in Action
Scenario Rule Triggered Decision
0.05 ETH to trusted payroll contract Whitelist 10× multiplier APPROVE
0.05 ETH to regular address Under autoApproveLimit (0.1 ETH) APPROVE
0.5 ETH to regular address Exceeds per-tx limit ESCALATE
Any amount to blacklisted address Blacklist hard-reject REJECT
0.05 ETH (daily cap reached) Would exceed 5 ETH/day ESCALATE
Rapid-fire tx burst Exceeds 20 tx/hour rate limit ESCALATE
Quickstart
// Initialize Warden with your policy config
import { Treasury } from './src/wallet/treasury.js'

const treasury = new Treasury({
  seed:   process.env.SEED_PHRASE,
  rpcUrl: process.env.RPC_URL,
  dbPath: './data/audit.db',
  policy: {
    autoApproveLimit: 10n ** 17n,        // 0.1 ETH per tx
    dailyLimit:       5n * 10n ** 18n,   // 5 ETH per day
    maxTxPerHour:     20,
    whitelist: ['0xPayrollContract...'],  // 10× limit
    blacklist: ['0xSuspiciousAddr...']   // always reject
  }
})

// Submit a transfer — policy evaluated before WDK signs
const result = await treasury.submit({
  to:     '0xRecipient...',
  value:  5n * 10n ** 16n,  // 0.05 ETH
  reason: 'Monthly subscription payment'
})

console.log(result.decision.decision)  // 'APPROVE' | 'REJECT' | 'ESCALATE'
